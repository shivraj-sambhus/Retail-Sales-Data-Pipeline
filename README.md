# Objective

In this project, I want to build an ETL pipeline of retail sales and weather data of an unspecified grocery store in Los Angeles.  To do this, I'll use Python and SQL. I'll also examine the relationship between the revenue and temperature of the stores by calculating the correlation between the two variables and by creating a heatmap visualization using the seaborn library. The main idea is that the revenue generated by each store is likely to be dependent on the weather during each day, potentially leading to increased in sales on hot days because of the need for cold items like beverages and ice creams.

# Extract

The given data is a set of daily retail sales of 10 grocery stores in LA. These files were provided in a variety of formats, such as .csv, .json, .parquet, and .sqlite. For the extract phase, I chose to use the .csv files and concatenated them in Pandas. 

Next, I used weather data from the OpenWeatherMap API for each day in LA and combined them in one dataset using the following FOR loop. The API key I used has since been deactivated and deleted.

```
# A list of weather data to be populated by the FOR loop below
weather_data = []

# Cleaning up the date column from the sales dataframe
dates = sales_df.date.sort_values().dropna().unique()

for date in dates:
    # The unix timestamp for each day of the month
    unix_timestamp = int(pd.Timestamp(date).timestamp()) 
    # The OpenWeatherMap API from which we pull the weather at each day
    url = f"https://api.openweathermap.org/data/3.0/onecall/timemachine?lat=34.0522&lon=-118.2437&dt={unix_timestamp}&appid={API_KEY}&units=imperial"
    
    # The weather data itself for the date
    response = requests.get(url)
    
    # Conditional loop that populates the weather_data list
    if response.status_code == 200:
        data = response.json()
        day_weather = {
            "date": pd.to_datetime(date).date(),
            "temp": data["data"][0]["temp"],
            "humidity": data["data"][0]["humidity"],
            "weather_main": data["data"][0]["weather"][0]["main"],
            "weather_desc": data["data"][0]["weather"][0]["description"]
        }
        weather_data.append(day_weather)
        print(f"Collected data for: {pd.to_datetime(date).date()}")
    else: 
        print(f"Failed to fetch data for {date}: {response.status_code}")
    
    # Respecting the API rate limit
    time.sleep(1) 

# The final dataframe
weather_df = pd.DataFrame(weather_data)
```

# Insights

The main insight is that the stores with the highest revenue (at least $60 per day) are the ones with high temperatures (96 degrees Farenheit to 104 degrees Farenheit) and low humidity (20% to 30%). This can be observed from the heatmap visualization above. The visualization includes data from the first store, LA001 but the data is similar for the remaining stores. Additionally, there is a weak positive linear correlation (0.19188) between the daily revenue of the stores and their temperature. This finding agrees with my initial assumption in the Objective section but the relationship between the two variables is likely nonlinear. This is because of the other confounding variables in the data, such as the humidity, proximity to other stores, and market conditions.
